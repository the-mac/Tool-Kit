sdkMulticast

September 26, 2011

This sample demonstrates how to communicate over a multicast group on a Windows Phone. It implements the classic Rock Paper Scissors game. Players join a multicast group to discover each other. UDP unicasting is then used to challenge a player and play the actual game. The UdpAnySourceMulticastClient class, available in Windows Phone OS 7.1, is used in this sample.

For more information about communicating using multicast on Windows Phone OS 7.1, see How to: Send and Receive Data in a Multicast Group for Windows Phone.

You will need to install Windows Phone SDK 7.1 to run this sample. To get started, go to App Hub.

To run the sample

  1. After unzipping the sample, note that there is one project. In the following instructions, we will explain how to start the Windows Phone client application.

    Note:

    To effectively demonstrate the multicast scenario, you should deploy this application to the emulator and to a device that is connected to a local Wi-Fi network.

  2. Double-click the .sln file to open the solution.

  3. Press F5 to start debugging the application.

  4. The Windows Phone client launches in the emulator. You should also deploy the application to your phone so that you have a minimum of two instances of the game in the multicast group.

  5. On all client devices where the application is running, enter a name in the Username field and click Join. Try to keep the names unique. There is no uniqueness checking in this sample, and using unique names will make it easier to understand the communication between clients. On each client, you will see a list of all clients that have joined the game (which is a multicast group).

  6. Select a name in the list on a device and click Challenge. This sends a challenge request to the player you selected. If the player accepts the challenge, the game will begin.

  7. The choice that each player makes is sent to the opponent’s device, and is displayed only after both of the players have selected a choice. The result of the round is then communicated to both players, and the score is updated accordingly.

  8. If you exit the game, this information is sent to all other multicast group members.
