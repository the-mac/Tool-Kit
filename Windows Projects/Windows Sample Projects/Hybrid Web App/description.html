This sample demonstrates how to quickly get up and running with a Windows Phone app that takes advantage of the effort you’ve already put into creating a great mobile website. This gives you an app presence in the Windows Phone Store with much less effort than developing a full-blown Windows Phone app while adding some features to your app that give it advantages over websites.

The sample app demonstrates how to:

  • Customize a Windows Phone app to showcase your mobile website.

  • Enable fast app resume to return the user to where they left off.

  • Handle the hardware back button for website navigation.

  • Pin specific webpages to the Windows Phone start screen with custom Tiles.

  • Share links to webpages on the user’s social media accounts.

Key APIs used in this sample:

  • WebBrowser

  • BackKeyPress

  • PhoneApplicationService

  • ShellTile

  • ShareLinkTask

  • OnNavigatedTo

  • OnNavigatedFrom

Build the sample

  1. Start Visual Studio Express 2012 for Windows Phone and select File >Open > Project/Solution.

  2. Go to the directory in which you unzipped the sample. Double-click the Visual Studio Express 2012 for Windows Phone Solution (.sln) file.

  3. Use Build > Rebuild Solution to build the sample

Run the sample

  1. To debug the app and then run it, press F5 or use Debug > Start Debugging. To run the app without debugging, press Ctrl+F5 or use Debug > Start Without Debugging.

  2. When the app launches, you see the Microsoft Store mobile website hosted in a browser control.

  3. The AppBar at the bottom of the screen contains buttons that implement home, refresh, pin to start, and share functionality.

  4. The pin to start button allows the user to pin the current page to Start, if it isn’t already pinned.

  5. The share button allows the user to share a link to the current page on the social media services they have enabled on their phone.

  6. The shopping cart menu item takes the user directly to their shopping cart, where they can complete the purchase process.
